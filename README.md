## Lab Question 2: SQL Database Implementation (`main.py`)

### Objective

Create tables from the relational schema, insert data, and run queries to list enrollments, resolve an instructor for a given course title, and count enrollments per course.

### Tech stack

- Python 3
- SQLite via the standard library [`sqlite3`](https://docs.python.org/3/library/sqlite3.html)

### Relational schema

| Table | Columns | Keys / constraints |
|-------|---------|-------------------|
| **Student** | `student_id`, `name`, `email`, `date_of_birth` | PK: `student_id`; `email` **UNIQUE** |
| **Instructor** | `instructor_id`, `name`, `department`, `phone_number` | PK: `instructor_id` |
| **Course** | `course_code`, `title`, `credit_hours`, `department`, `instructor_id` | PK: `course_code`; FK: `instructor_id` → `Instructor`; `instructor_id` **NOT NULL** |
| **Enrollment** | `student_id`, `course_code` | Composite PK `(student_id, course_code)`; FKs to `Student` and `Course` |

Foreign keys are enforced with `PRAGMA foreign_keys = ON` on the connection.

### Data seeding (summary)

Each run **drops and recreates** all tables, then inserts at least **20 rows per table** where applicable:

- **Students:** real registration numbers / names / emails plus generated rows to reach 20.
- **Instructors:** 20 rows with distinct names (`INS001` … `INS020`).
- **Courses:** 20 courses including a **`Database Systems`** row (for Query B) plus real course titles/codes from the lab list; `course_code` values are normalized (spaces removed, e.g. `COS 201` → `COS201`).
- **Enrollment:** 20 rows (one enrollment per student), distributed across the 20 courses.

### Required queries (implemented in `main.py`)

1. **Query A — Students and enrolled courses**  
   Join `Student` → `Enrollment` → `Course` and list each student’s enrolled course title.

2. **Query B — Instructor for “Database Systems”**  
   Join `Instructor` → `Course` where `Course.title = 'Database Systems'`.

3. **Query C — Count students per course**  
   `LEFT JOIN` `Course` with `Enrollment`, `GROUP BY` course, `COUNT` enrollments (courses with zero students still appear).

Equivalent SQL (for reference):

```sql
-- Query A
SELECT Student.name, Course.title
FROM Student
JOIN Enrollment ON Student.student_id = Enrollment.student_id
JOIN Course ON Enrollment.course_code = Course.course_code;

-- Query B
SELECT Instructor.name
FROM Instructor
JOIN Course ON Instructor.instructor_id = Course.instructor_id
WHERE Course.title = 'Database Systems';

-- Query C
SELECT Course.course_code, Course.title,
       COUNT(Enrollment.student_id) AS students_enrolled
FROM Course
LEFT JOIN Enrollment ON Enrollment.course_code = Course.course_code
GROUP BY Course.course_code, Course.title
ORDER BY Course.course_code;
```

### How to run

```bash
cd introduction_to_data_management
python3 main.py
```

Each execution rebuilds `miva_university.db` from scratch.

## 5NF HR database (`miva_database_management.py`)
